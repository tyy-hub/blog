Git

基本操作

1. 拉取项目

  • git clone 地址
  • npm install
  • 若有子仓库则需要使用 git submodule init git submodule update 拉取子仓库

2. 上传代码

  • git add .
  • git commit -m'[*修改走查样式]'
  • git pull
  • git push

3. 分支

  • 创建新分支

    • 在当前分支创建新分支 git checkout -b 2.0.10
    • 提交 git push --set-upstream origin 2.0.10

  • 切换分支 git checkout 2.0.10

  • 合并分支

    • 在当前分支git merge 2.0.10 将2.0.10分支合并到了当前分支
    • git push

  • 分支重命名

    • git branch -m oldName newName 重命名远程分支对应的本地分支
    • git branch --delete origin oldName 删除远程分支
    • git push origin newName 上传新命名的本地分支
    • git branch --set-upstream-to origin/newName 把修改后的本地分支与远程分支关联

4. 回退版本

  • 查看提交记录 git log
  • 回退到上一个版本 git reset --hard HEAD^
  • 回退到指定版本 git reset --hard commitID
  • 提交 git push origin 2.0.10 --force
  • 回退到merge前 git reset --merge

5. 删除分支

删除分支需要分为两步操作：

  • 删除远程分支 git push origin --delete [branch_name]
  • 删除本地分支 git branch -D [branch_name]

6. 将一个分支的某次commit拉取到当前分支

git cherry-pick (commitID)
git push

7. 查看日志

gitk --all

8. 缓存

  • 缓存代码 git stash save
  • 查看缓存列表 git stash list
  • 取消缓存 git stash pop

9. 查看当前分支从哪拉取的

git reflog --date=local | grep 分支名

Vue

Vue的生命周期

1. 创建

  • beforeCreate
    数据观测(data observer)和 event/watcher 事件配置之前被调用

  • created(常用)
    data 设置完成,可以修改了。在此处可以执行修改 data 的操作,也就是以后的异步请求修改数据可以在此生命周期执行。

2. 挂载

  • beforeMount
    挂载之前,在此之前虚拟 DOM 已经创建完成,组件的 render 函数首次执行,

  • mounted(常用)
    组件完全渲染到了页面中。可以获取真实的 dom 节点

3. 更新

  • beforeUpdate
    数据更新时

  • updated(常用)
    数据更新完毕执行,页面更新也完毕了。可以获取更新之后的 data 以及真实 dom 节点

4. 销毁

  • beforeDestroy
    组件销毁之前, 手动撤销监听事件，计时器等

  • destroyed
    组件销毁完毕，此时仅存在 Dom 节点，其他东西已自动销毁

常用知识点

  • created 里可以通过 setTimeout 异步修改 data，异步修改完之后只有 updated 能够取到,且需要通过 v-if 与页面挂勾
  • created 里可以同步修改 data，同步修改完之后 mounted 也可取到(但若写了 v-if 则取不到)
