Vue的生命周期

1. 创建

  • beforeCreate
    数据观测(data observer)和 event/watcher 事件配置之前被调用

  • created(常用)
    data 设置完成,可以修改了。在此处可以执行修改 data 的操作,也就是以后的异步请求修改数据可以在此生命周期执行。

2. 挂载

  • beforeMount
    挂载之前,在此之前虚拟 DOM 已经创建完成,组件的 render 函数首次执行,

  • mounted(常用)
    组件完全渲染到了页面中。可以获取真实的 dom 节点

3. 更新

  • beforeUpdate
    数据更新时

  • updated(常用)
    数据更新完毕执行,页面更新也完毕了。可以获取更新之后的 data 以及真实 dom 节点

4. 销毁

  • beforeDestroy
    组件销毁之前, 手动撤销监听事件，计时器等

  • destroyed
    组件销毁完毕，此时仅存在 Dom 节点，其他东西已自动销毁

常用知识点

  • created 里可以通过 setTimeout 异步修改 data，异步修改完之后只有 updated 能够取到,且需要通过 v-if 与页面挂勾
  • created 里可以同步修改 data，同步修改完之后 mounted 也可取到(但若写了 v-if 则取不到)
