组件之间的通信

vue 组件之间通信的 9 种方式

  • props 和$emit(常用)
  • .sync
  • v-model
  • slot
  • 定义 ref
  • vuex
  • $parent和$children
  • provide 和 inject
  • $attrs和$listeners

  1. props 和$emit

    父组件向子组件传递数据是通过 prop 传递的，子组件传递数据给父组件是通过$emit 触发事件来做到的

    • props

    • $emit

  2. .sync

    sync 修饰符，与我们平常使用$emit 实现父子组件通信没有区别，只不过是写法上方便一些。

  3. v-model

    父组件通过 v-model 传递值给子组件时，会自动传递一个 value 的 prop 属性，在子组件中通过 this.$emit(‘input’,val)自动修改 v-model 绑定的值

  4. slot

    父传子

    • 普通插槽

    • 具名插槽

  5. 定义 ref

    在父组件调用子组件的时候直接添加 ref 属性, 即可拿到子组件的属性、方法、data 等内容

  6. vuex

    如果业务逻辑复杂，很多组件之间需要同时处理一些公共的数据，这个时候才有上面这一些方法可能不利于项目的维护，vuex 的做法就是将这一些公共的数据抽离出来，然后其他组件就可以对这个公共数据进行读写操作，这样达到了解耦的目的。

  7. $parent和$children

    • $parent 可以在子组件内通过 this.$parent.属性名/方法名 取到父组件

    • $children 可以在父组件内通过 this.$children.属性名 取到子组件

  8. provide 和 inject

    • provice

    • inject

  9. $attrs 和 $listeners

    • $attrs 获取当前组件的所有 html 属性, 即父组件使用我的时候标签内传递的所有属性(class 除外)

    • $listeners 获取当前组件的所有 html 方法, 即父组件使用我的时候标签内传递的所有方法

Vue

开发中常用的指令有哪些

  • v-model 多用于表单元素实现双向数据绑定（同 angular 中的 ng-model）

  • v-html：更新元素的 innerHTML

  • v-show 显示内容 （同 angular 中的 ng-show）

  • v-hide 隐藏内容（同 angular 中的 ng-hide）

  • v-if 显示与隐藏 （dom 元素的删除添加 同 angular 中的 ng-if 默认值为 false）

  • v-on:click：可以简写为@click，@绑定一个事件，如果事件触发了，就可以指定事件的处理函数，函数必须写在 methods 里面

  • v-bind 动态绑定 作用： 及时对页面的数据进行更改

  • v-bind:title=’msg’ 简写：title=’msg’

  • v-bind:class 三种绑定方法 1、对象型 ‘{red:isred}’ 2、三元型 ‘isred?”red”:”blue”‘ 3、数组型 ‘[{red:”isred”},{blue:”isblue”}]’

  • v-for 格式： v-for=”字段名 in(of) 数组 json” 循环数组或 json(同 angular 中的 ng-repeat)

  • v-else-if 必须和 v-if 连用

  • v-else 必须和 v-if 连用 不能单独使用 否则报错 模板编译错误

  • v-text 解析文本

  • v-html 解析 html 标签

  • v-once 进入页面时 只渲染一次 不在进行渲染

  • v-cloak 防止闪烁

  • v-pre 把标签内部的元素原位输出

v-show与v-if

v-show

  • v-show 是通过设置 DOM 元素的 display 样式属性控制显隐

  • v-show 只是简单的基于 css 切换

  • 在任何条件下（首次条件是否为真）都被编译，然后被缓存，而且 DOM 元素保留

  • v-show 有更高的初始渲染消耗

  • v-show 适合频繁切换

v-if

  • v-if 是动态的向 DOM 树内添加或者删除 DOM 元素

  • v-if 切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件

  • v-if 是惰性的，如果初始条件为假，则什么也不做；只有在条件第一次变为真时才开始局部编译（编译被缓存？编译被缓存后，然后再切换的时候进行局部卸载)

  • v-if 有更高的切换消耗

  • v-if 适合运营条件不大可能改变时

watch与计算属性

  • 计算属性 computed

    • 支援缓存，只有依赖数据发生改变，才会重新进行计算

    • 不支持异步

    • 如果一个属性是由其他属性计算而来的，这个属性依赖其他属性，是一个多对一或者一对一，一般用 computed

    • 如果 computed 属性属性值是函数那么默认会走 get 方法；函数的返回值就是属性的属性值；在 computed 中的属性都有一个 get 和 set 方法，当数据变化时，调用 set 方法

  • 侦听属性 watch

    • 不支持缓存，只要数据变就直接触发相应的操作

    • watch 支持异步

    • 监听的函数接收两个参数，第一个参数是最新的值，第二个参数是之前的值

    • 监听数据必须是 data 中声明过或者父组件传递过来的 props 中的数据，当数据变化时，触发其他操作，函数有两个参数：

      immediate：组件加载立即触发回调函数执行

      deep: 深度监听，为了发现对象内部值的变化，复杂类型的数据时使用，例如数组中的对象内容的改变，注意监听数组的变动不需要这么做。注意：deep 无法监听到数组的变动和对象的新增，参考 vue 数组变异,只有以响应式的方式触发才会被监听到。

Vue

test1

1. 创建

  • beforeCreate
    数据观测(data observer)和 event/watcher 事件配置之前被调用

  • created(常用)
    data 设置完成,可以修改了。在此处可以执行修改 data 的操作,也就是以后的异步请求修改数据可以在此生命周期执行。

2. 挂载

  • beforeMount
    挂载之前,在此之前虚拟 DOM 已经创建完成,组件的 render 函数首次执行,

  • mounted(常用)
    组件完全渲染到了页面中。可以获取真实的 dom 节点

3. 更新

  • beforeUpdate
    数据更新时

  • updated(常用)
    数据更新完毕执行,页面更新也完毕了。可以获取更新之后的 data 以及真实 dom 节点

4. 销毁

  • beforeDestroy
    组件销毁之前, 手动撤销监听事件，计时器等

  • destroyed
    组件销毁完毕，此时仅存在 Dom 节点，其他东西已自动销毁

常用知识点

  • created 里可以通过 setTimeout 异步修改 data，异步修改完之后只有 updated 能够取到,且需要通过 v-if 与页面挂勾
  • created 里可以同步修改 data，同步修改完之后 mounted 也可取到(但若写了 v-if 则取不到)
